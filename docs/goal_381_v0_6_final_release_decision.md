# Goal 381: v0.6 final release decision

## Why this goal exists

After the final external release review, the repo needs one explicit decision
goal that says whether the current `v0.6` package is actually ready to become
`v0.6.0`.

## Scope

In scope:

- synthesize the final release-facing evidence
- state whether `v0.6.0` should be made now
- preserve the final honest release boundary

Out of scope:

- the mechanical act of tagging or pushing
- unrelated repo cleanup

## Exit condition

This goal is complete when the repo has:

- a saved final release-decision report
- a saved review note
- a saved Codex consensus note
- a clear accept/reject decision on making `v0.6.0`
