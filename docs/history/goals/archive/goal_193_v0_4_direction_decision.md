# Goal 193: v0.4 Direction Decision

## Purpose

Decide what `v0.4` should be after the `v0.3.0` release.

The decision must be grounded in the current front-door story and must answer a
hard product question:

- should RTDL lean further into application-style demos
- or should RTDL convert the new `v0.3.0` proof work into a stronger
  non-graphical workload and language surface

## Required output

- one detailed proposal/rebuttal report
- explicit competing options
- explicit conflicts and rebuttals
- one recommended `v0.4` direction
- Claude review
- Gemini review
- Codex consensus

## Scope rules

- use the current front page and current release docs as the grounding surface
- keep the RTDL honesty boundary intact:
  - RTDL is a geometric-query runtime
  - RTDL is not a general-purpose renderer
- prefer a direction that strengthens RTDL's long-term language/runtime
  identity rather than a direction that only makes the visual demo prettier
