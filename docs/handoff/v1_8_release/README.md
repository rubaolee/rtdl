# v1.8 Release Handoff Notes

This folder holds operational handoff notes, audit plans, and pod-validation
next-step notes produced during the v1.8 release-prep and release cycle.

They are intentionally kept out of the repository root so the GitHub front page
stays focused on user-facing project files:

- `README.md`
- `VERSION`
- `requirements.txt`
- `Makefile`
- `.gitignore`

For durable release evidence, prefer the formal reports under
`docs/reports/`, release packages under `docs/release_reports/`, and reviews
under `docs/reviews/`.
