# Goal 252: Archive And Early Goal Docs Audit Pass

## Objective

Expand tier-3 audit coverage through the archive entrypoints and the early
v0.1/v0.2 goal-definition documents without flattening historical planning into
fake current-state docs.

## Scope

This pass covers:

- `docs/archive/README.md`
- `docs/archive/v0_1/README.md`
- `docs/goal_100...goal_111`

## Required Checks

- archive entrypoints distinguish clearly between archived releases and the live released line
- early goal docs remain historically honest and understandable
- historical goal docs are treated as planning/closure records, not rewritten into current product docs
- easy-to-reach archive surfaces do not misdirect users away from the current `v0.4.0` release surface
