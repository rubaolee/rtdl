Purpose:
- close a release-gate style audit for the current `v0.5` line
- answer whether the repo is honestly ready for a `v0.5 preview` claim surface
- distinguish preview readiness from final release sign-off

Success criteria:
- write one current readiness audit report under `docs/reports/`

  - what is still missing for a final `v0.5` release
- keep the backend/platform honesty boundary explicit
- make a clear decision between:
  - preview-ready
  - not preview-ready
  - final-release-ready
  - not final-release-ready

- Codex consensus note saved in the repo
