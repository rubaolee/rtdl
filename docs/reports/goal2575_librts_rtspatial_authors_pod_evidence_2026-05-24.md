# Goal2575 LibRTS Authors-Code Pod Evidence

## Scope

This goal moves the LibRTS benchmark from local intake to first authors-code
evidence. It builds and runs the public `RTSpatial/RTSpatial` implementation for:

- point query: indexed box contains query point;
- range-contains query: indexed box contains query box;
- range-intersects query: indexed box intersects query box.

Paper:

- Liang Geng, Rubao Lee, and Xiaodong Zhang, "LibRTS: A Spatial Indexing Library by Ray Tracing," PPoPP 2025.
- DOI: `10.1145/3710848.3710850`
- Authors code: `https://github.com/RTSpatial/RTSpatial`

## Build Result

The authors executable `rtspatial_exec` builds successfully with:

```bash
cmake -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTS=OFF \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.6/bin/nvcc \
  -DCUDAToolkit_ROOT=/usr/local/cuda-12.6 \
  -DCMAKE_CUDA_ARCHITECTURES=86 \
  -DOptiX_INSTALL_DIR=/opt/optix/NVIDIA-OptiX-SDK-8.1.0-linux64-x86_64 ..
cmake --build . -j$(nproc)
```

CUDA 12.8 failed at runtime with:

```text
cudaErrorUnsupportedPtxVersion: the provided PTX was compiled with an unsupported toolchain.
```

Reason: the pod driver is `565.57.01` and advertises CUDA 12.7 support, while
CUDA 12.8 emitted newer PTX. Rebuilding with CUDA 12.6 fixed the issue.

## Interpretation

This establishes that the authors' LibRTS implementation is buildable and
runnable on the pod, and that the RTDL WKT fixture generator preserves the
authors' predicate semantics for the checked 10k x 1k row.

The timing ratios are internally consistent: range-intersects is about 2x the
query time of point/range-contains at 100k and 1M because the authors path uses
the more complex forward/backward range-intersection traversal.

## Claim Boundary

- This is authors-code timing evidence, not an RTDL performance claim.
- The 10k row is same-fixture correctness evidence against RTDL's CPU oracle.
- The 100k and 1M rows are authors-code timing rows with counts recorded; CPU
  oracle verification was intentionally skipped to avoid O(n*m) Python cost.
- These are synthetic uniform fixtures generated by RTDL, not the full paper
  artifact datasets.
- This does not reproduce the paper's headline speedups yet.
- This does not add a native RTDL engine primitive.
- No public speedup wording is authorized by this report.
