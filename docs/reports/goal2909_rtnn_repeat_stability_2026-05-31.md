# Goal2909: RTNN Repeat Stability

Date: 2026-05-31
Status: implemented with pod evidence

## Purpose

Goal2908 reran the full v2.5 packet after the Hausdorff repeat-stability fix. Hausdorff left the target list, but RTNN reappeared with one distribution-dependent packet row:

- `clustered`: `0.945x` CuPy/RTDL in a repeat-3 packet median
- `uniform`: `0.980x` CuPy/RTDL
- `shell`: `1.847x` CuPy/RTDL

The clustered row contradicted earlier RTNN evidence and had a large first-run wobble in the RTDL samples. Goal2909 checks whether this is a real v2.5 primitive problem or another short-row timing artifact.

## Probe

Artifact directory:

`docs/reports/goal2909_rtnn_repeat_stability_pod/`

The probe reuses the exact `clustered_32768.csv` generated by the Goal2908 packet and runs repeat-9 timing for:

- current prepared-query aggregate mode
- prepared-query batch mode
- prepared-query batch graph mode
- CuPy grid same-contract baseline

Measured medians:

| Path | Median sec | Candidate count | Result |
| --- | ---: | ---: | --- |
| RTDL prepared-query aggregate | `0.004530` | `1,287,932` | pass |
| RTDL prepared-query batch | `0.004645` | `1,287,932` | pass |
| RTDL prepared-query batch graph | `0.004526` | `1,287,932` | pass |
| CuPy grid | `0.011978` | `1,287,932` | pass |

The stable repeat-9 result is about `2.65x` faster than the CuPy grid same-contract baseline on the same clustered input.

## Change

The RTNN canonical harness now defaults to `repeat = 9`, matching the evidence requirement for short rows whose median can be distorted by one outlier.

No native engine code changed. No RTNN-specific native behavior was added.

## Interpretation

The current RTNN issue was not a missing primitive. It was a measurement stability problem in a very short row. The current prepared-query aggregate path is already competitive, and the graph path is marginally fastest on the clustered probe.

The broader design lesson still holds: serious benchmark apps need stable measurement policy as much as faster kernels. A 3-run median can turn a winning path into a false target when one launch sample is noisy.
