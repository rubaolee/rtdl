# Goal2578 LibRTS Paper-Like Uniform Authors-Code Evidence

## Scope

Goal2575 used valid RTDL-generated uniform fixtures, but their boxes were much
larger than the authors artifact generator's default small-box scale. This goal
adds a paper-like uniform slice using:

- max indexed-box width/height: `0.005`
- max query-box width/height: `0.005`
- 1k point queries and 1k box queries
- authors `rtspatial_exec`

The fixtures are still generated by RTDL's WKT generator, not by the authors
generator, so this is paper-like evidence rather than exact paper artifact
reproduction.

## Selectivity Insight

Goal2575's 1M x 1k high-selectivity fixture used much larger indexed boxes and
query boxes. On the same authors code and pod, it produced much larger result
sets and much slower query times:

| 1M x 1k fixture | Point query | Range contains | Range intersects |
| --- | ---: | ---: | ---: |
| high-selectivity Goal2575 fixture | `218.040 ms` | `216.012 ms` | `423.462 ms` |
| paper-like small-box Goal2578 fixture | `0.099 ms` | `0.106 ms` | `0.620 ms` |

Main conclusion: LibRTS query time is highly workload/selectivity dependent.
The paper-like small-box scale is orders of magnitude lighter than the earlier
stress fixture. Future RTDL comparisons must report fixture sizes and result
counts next to timing, otherwise the numbers are misleading.
