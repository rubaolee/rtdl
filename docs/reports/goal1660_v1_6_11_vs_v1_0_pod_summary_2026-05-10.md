# Goal1660 v1.6.11 vs v1.0 Pod Performance Summary

## Verdict

`measured_with_scope_correction_not_release_ready`

The pod run produced accepted cross-version timing for the rows that are genuinely runnable under the current release-facing app profilers. It also proved that most requested Embree app rows are not currently supported by those profilers; they must be reported as unsupported, not as slow/fail performance results.

## Interpretation

The measured OptiX rows are mostly close to v1.0 wall-clock time at this benchmark scale, but they do not support a public speedup claim. Several rows are slower in v1.6.11 than v1.0, with the largest measured ratio in this package at `1.138` for `segment_polygon_anyhit_rows/optix`.

The database app is the only row in this package with both Embree and OptiX cross-version timing because it exposes a real `--backend embree|optix` selector. For most other app profilers, the current release-facing scripts are OptiX-specific; adding true Embree-comparable app profilers is separate implementation work, not a pod measurement problem.

## Release Boundary

This package blocks release/public speedup claims. It is valid measured evidence for the supported rows, but it is not enough to tag or publish v1.6.11 as a performance release.
