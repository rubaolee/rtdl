# Goal2331 RayJoin Query-Exec Export Patch Plan

Date: 2026-05-18

Status: `patch-drafted-build-not-validated`

## Purpose

Goal2329 proved that RTDL can consume deterministic same-query streams, but the
streams were generated by RTDL's local demo generator. That is useful for
runtime validation, but it is not the same as RayJoin paper evidence.

The next pod round needs streams produced by RayJoin's own `query_exec`, using
the exact generated LSI/PIP query points and segments that RayJoin times. This
report records the source inspection and the draft patch prepared while no pod
was available.

Important finding: RTDL's demo stream generator is intentionally not the same
as RayJoin's generator. RayJoin's LSI generator samples two random map-bounds
points, normalizes the direction, then samples length `t` uniformly from
`[0, gen_t]`. The RTDL demo stream used a dependency-free local smoke generator.
Therefore, serious same-contract evidence must export from RayJoin itself.

## Draft Patch

Draft patch path:

`docs/research/rayjoin_query_exec_export_patch.diff`

The patch adds:

- a new `query_exec` flag: `-export_query_stream=/path/to/stream.json`;
- a matching `QueryConfig::export_query_stream` field;
- JSON export from RayJoin's own `GenerateLSIQueries(...)`;
- JSON export from RayJoin's own `GeneratePIPQueries(...)`;
- RTDL-compatible schema: `rtdl.rayjoin.same_query_stream.v1`;
- producer marker: `rayjoin_query_exec_export_patch`;
- claim boundary fields that keep public claims locked.

The intended RayJoin command shape is:

```bash
./query_exec \
  -poly1 base_map.cdb \
  -mode=rt \
  -query=lsi \
  -gen_n=65536 \
  -gen_t=0.1 \
  -seed=2327 \
  -warmup=5 \
  -repeat=15 \
  -export_query_stream=/tmp/rayjoin_lsi_stream.json
```

and similarly for PIP:

```bash
./query_exec \
  -poly1 base_map.cdb \
  -mode=rt \
  -query=pip \
  -gen_n=65536 \
  -gen_t=0.1 \
  -seed=2328 \
  -warmup=5 \
  -repeat=15 \
  -export_query_stream=/tmp/rayjoin_pip_stream.json
```

Those stream files should then be passed to RTDL:

```bash
PYTHONPATH=src:. \
RTDL_OPTIX_LIBRARY=$PWD/build/librtdl_optix.so \
LSI_STREAM=/tmp/rayjoin_lsi_stream.json \
PIP_STREAM=/tmp/rayjoin_pip_stream.json \
bash scripts/goal2327_rayjoin_pod_perf_runner.sh
```

## Local Limitation

The local Linux host did not have the RayJoin build dependencies installed:

- `libgoogle-glog-dev`: absent;
- `libgflags-dev`: absent.

So this patch is drafted, `git diff --check` clean, and `git apply --check`
clean against upstream commit `02bf622`. It is not yet compiled or run against
upstream RayJoin. Build validation should happen on the next pod or on a host
where RayJoin dependencies are available.

## Why This Matters

This patch is the difference between:

- RTDL consuming a deterministic demo stream, which is good for app/runtime
  validation; and
- RTDL consuming a RayJoin-authored stream, which is required for a serious
  same-contract comparison against RayJoin `query_exec`.

Without this export step, any RTDL-vs-RayJoin comparison remains weaker because
the query stream could differ from RayJoin's actual generated workload.

## Claim Boundary

This plan does not authorize:

- RTDL-vs-RayJoin speedup claims;
- RayJoin paper reproduction claims;
- v2.0 release claims.

It only prepares the input-stream bridge needed for the next pod evidence round.
