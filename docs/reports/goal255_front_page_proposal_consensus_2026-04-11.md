## Converged Direction

All three proposal legs agreed on the same primary diagnosis:

- the current front page is overloaded
- the live release identity is not prominent enough
- the first-run path is too buried
- historical and internal detail appears too early
- the visual demo is explained honestly but too defensively

## Agreed Rewrite Shape

The final front page should:

1. lead with a one-screen identity for RTDL
2. put the fastest safe first-run path near the top
3. present the video/demo as proof of versatility, not as a renderer claim
4. clearly state that the current release is `v0.4.0`
5. list the live workload surface accurately, including the `v0.4` nearest-neighbor additions
6. move deeper research/history detail lower on the page or out to the docs index

## Guardrails

- keep RTDL framed as a language/runtime for non-graphical geometric-query work
- do not overclaim backend availability on the front page
- keep honest limits visible, but not above the first practical user path
- do not send normal users into archive/history material from the front door

## Implementation Decision

The rewrite should follow this front-page order:

1. RTDL identity
2. See it quickly
3. Start in two minutes
4. Choose your path
5. Current release state
6. What RTDL contains
7. Why RTDL exists
8. Current limits
9. Repository layout / docs links

That is the consensus direction for the final implementation.
