## Findings

**1. Compared paths and matched inputs (technically sound)**

Both native executables write CSV inputs generated by the same `write_segments_csv`/`write_points_csv`/`write_polygons_csv` helpers used for the RTDL calls. The FNV-64 hash is implemented identically in Python (`_fnv64_from_pairs`, `scripts/goal19_compare_embree_performance.py:41-48`) and C++ (`hash_pairs`, `apps/goal15_lsi_native.cpp:112-126`, `apps/goal15_pip_native.cpp:153-167`) — same constants, same sort-before-hash order. The pip `contains==1` filter is applied consistently in both the native executable (`apps/goal15_pip_native.cpp:227`) and the Python hash helper (`scripts/goal19_compare_embree_performance.py:55`). Inputs are genuinely matched.

**2. Runtime budget**

Reported total wall time is `8.74 min`. This is within the agreed 5–10 minute window.

**3. Correctness checks**

Fixture section verifies all three: `dict_matches_native`, `raw_matches_dict`, `prepared_raw_matches_dict` — all `True` for both lsi and pip before any timing claims are accepted. Large-profile section uses hash-based `dict_matches_native` only, which is sufficient given the fixture section already establishes three-way parity at smaller scale. The test (`tests/goal19_compare_test.py`) covers `dict_matches_native` and `raw_matches_dict` for both sections; it omits `prepared_raw_matches_dict` from its assertions, but the harness records it and the actual run shows `True`.

**4. Honesty about dict vs raw/prepared performance**

The report does not soften the dict-path result: 13–225x slower than native across workloads and scales. The raw and prepared-raw gaps (0.51–1.82x at fixture, 0.83–0.98x at large scale) are stated plainly. The implementation report explicitly says "the ordinary dict-return RTDL path is still nowhere near native performance" and recommends treating it as a convenience path only. The README repeats the same framing. The plan's criterion 6 ("state clearly whether RTDL is still materially slower / close enough / still in need of another runtime redesign slice") is satisfied — the answer given is that raw/prepared paths are close enough and the dict path is not.

**5. Blockers**

None. All six acceptance criteria from the plan are met. The minimum scope (lsi + pip) is covered. Native baselines are claimed only for workloads with actual executables. The test suite passes (80 tests).

Goal 19 delivers what the plan required: matched inputs, three-way correctness parity confirmed at fixture scale, hash-verified correctness at large scale, a total run inside the 5–10 minute budget, and an honest characterization of all four runtime paths.
