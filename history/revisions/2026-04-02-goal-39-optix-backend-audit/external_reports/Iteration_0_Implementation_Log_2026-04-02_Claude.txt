  OptiX Backend — What was implemented

  src/native/rtdl_optix.cpp (~950 lines)

  The C++ host runtime. Key design:

  - Device kernels as embedded string literals — compiled at runtime via NVRTC, so no offline nvcc step is needed for the device
  code.
  - Lazy pipeline caching (std::call_once) — the NVRTC + OptiX JIT cost is paid only once per workload type per process.
  - All-hits traversal — LSI, PIP, overlay, ray-hitcount, and seg-poly-hitcount use the __intersection__ + __anyhit__ pattern with
  optixIgnoreIntersection() so every valid hit is reported, not just the closest.
  - Output strategies per workload:
    - LSI: atomic counter + dynamic output buffer (upper-bounded at N×M)
    - PIP / Overlay: pre-allocated N×M output array, updated in-place by anyhit
    - RayHitCount / SegPolyHitcount: per-probe hit counter in payload register p1, no global atomics
    - PointNearestSegment: plain CUDA kernel (brute-force parallel), not OptiX — distance queries don't map to ray traversal
  - Data precision: host inputs/outputs stay double; GPU works in float32 (matching the float_approx precision mode).
  - Overlay GPU/CPU split: GPU handles the LSI (edge–edge) component via BVH; CPU supplements the PIP (vertex-in-polygon) component
   after the launch.
